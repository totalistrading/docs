---
title: 'MM Heartbeat'
description: 'Send a liveness heartbeat to the Quote Service. Active MMs should send heartbeats at least once per minute.'
---

```
POST /v1/mm/heartbeat
```

Sends a liveness heartbeat for the connected market maker. The Quote Service records heartbeats to track MM availability. Heartbeat status is currently informational; quote exclusion based on liveness is not yet enforced but is planned.

**Auth:** `X-API-Key` (market maker)

**Base URL:** Quote Service (configured per environment)

## Request Body

No request body required.

## Response

**`200 OK`**

```json
{
  "data": {
    "status": "ok",
    "mm_id": "mm-uuid",
    "ttl_ms": 60000
  }
}
```

| Field | Type | Description |
| --- | --- | --- |
| `status` | string | Always `"ok"`. |
| `mm_id` | string | The MM's user ID. |
| `ttl_ms` | integer | The heartbeat TTL in milliseconds. Send the next heartbeat before this expires. |

## Recommended Cadence

Send a heartbeat every **30 seconds** to stay well within the 60-second TTL. A simple interval loop alongside the SSE stream connection works well:

```javascript
setInterval(() => {
  fetch('/v1/mm/heartbeat', {
    method: 'POST',
    headers: { 'X-API-Key': API_KEY },
  });
}, 30_000);
```
