Totalis supports two integration models. Both expose the same trading surface — markets, quoting, and settlement — and differ only in who holds the signing keys for the underlying Solana wallet.

| | Managed (custodial) | Headless (non-custodial) |
| --- | --- | --- |
| Status | Available today | In development |
| Keys held by | Totalis-managed wallet | You or your end-users |
| Onboarding | One-time delegation | Fully programmatic |
| Best for | Individuals, quick clients | Platforms that custody their own users |

## Choosing a Model

**Managed** is the fastest path. Totalis provisions and custodies an embedded wallet; you fund it and trade with an API key. The only non-programmatic step is a one-time delegation that authorizes Totalis to sign on the wallet's behalf.

**Headless** keeps custody with you. You, or your end-users, hold the keys. Totalis builds each transaction, you sign it with your own custody stack, and Totalis broadcasts and settles. There is no delegation and no browser step at any point. A single API principal manages any number of sub-accounts, one per end-user, each with its own vault. This is the model for partners running their own platform.

Both models share the same base URL and authentication:

- Base URL: `https://api.totalis.trade`
- Authentication: an API key sent in the `X-API-Key` header on every request.
