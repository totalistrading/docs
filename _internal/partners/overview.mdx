Totalis offers two integration models. They share the same trading surface (markets, quoting, settlement) and differ only in **who holds the signing keys**.

| | **Managed (custodial)** | **Headless (non-custodial)** |
|---|---|---|
| **Status** | ✅ Live today | 🚧 In development (preview) |
| **Keys held by** | Totalis-managed embedded wallet | **You / your users** |
| **Onboarding** | One-time delegation (interactive consent) | Fully headless — no browser, ever |
| **Best for** | Individuals; quick programmatic trading | Platforms with their own users + custody stack |
| **Guide** | [Managed Quickstart](/_internal/partners/quickstart-managed) | [Headless Integration](/_internal/partners/headless-integration) |

## Which one?

- **Managed** — Totalis provisions and custodies an embedded wallet; you fund it and trade via API key. Simplest to start. The only non-headless step is a one-time "enable trading" delegation.
- **Headless** — you (or your end-users) hold the keys; Totalis builds transactions, you sign them with your own custody stack (Privy, Turnkey, Fireblocks, raw keypair — signer-agnostic), and Totalis broadcasts + settles. No delegation, no custody transfer, no browser. One API principal fans out to **N sub-accounts**, one per end-user. This is the model for partners who run their own platform.

Both use the same base URL and auth:

- **Base URL:** `https://api.totalis.trade`
- **Auth:** `X-API-Key` header on every request.
