The managed model provisions a Totalis-custodied embedded wallet that you fund on Solana and trade with an API key. This guide covers authentication, market data, quoting, and funds.

- Base URL: `https://api.totalis.trade`
- Full reference: [docs.totalis.trade](https://docs.totalis.trade)
- Authentication: an API key in the `X-API-Key` header.

<Note>
  Trading requires a one-time delegation that authorizes Totalis to sign on the wallet's behalf. Today that step is completed interactively in the Totalis app and cannot yet be done through the API alone. Partners that need fully programmatic onboarding should use the [headless model](/_internal/partners/headless-integration).
</Note>

## Authentication

Generate an API key from the [dashboard](https://app.totalis.trade) under Account → API Keys. The key is shown once; store it securely.

```bash
curl https://api.totalis.trade/v1/me -H "X-API-Key: <your-api-key>"
```

Keys are scoped. Request only the scopes you need: `markets:read`, `trading:write`, `balances:read`, `funds:withdraw`, `account:read`, `account:write`.

## Trade Lifecycle

<Steps>
  <Step title="Fund the wallet">
    Send native USDC to your embedded wallet address (see [Funds](#funds)).
  </Step>
  <Step title="Browse markets">
    List markets and check which legs can be combined.
  </Step>
  <Step title="Open a quote request">
    Submit a parlay of 2–5 legs; market makers compete to price it.
  </Step>
  <Step title="Commit a quote">
    Committing the chosen quote creates the position on-chain.
  </Step>
  <Step title="Settle and withdraw">
    Positions settle automatically at resolution; withdraw when ready.
  </Step>
</Steps>

## Account

| Endpoint | Purpose |
| --- | --- |
| `GET /v1/me` | Profile and identifiers |
| `GET /v1/wallet` | Embedded wallet address and balances |
| `GET /v1/portfolio` | Consolidated balance, stats, and position counts |
| `GET /v1/api-keys`, `POST /v1/api-keys`, `DELETE /v1/api-keys/{prefix}` | Manage API keys (creation requires a dashboard session) |

## Markets

| Endpoint | Purpose |
| --- | --- |
| `GET /markets` | List tradeable markets |
| `GET /markets/{ticker}` | Market detail |
| `GET /markets/exclusion-groups` | Leg combinations blocked by correlation rules |

`market_ticker` is venue-specific. For Kalshi, use the Kalshi ticker (for example `KXBTC-25FEB07-T100000`). For Polymarket, use the `condition_id` (0x hex) together with `side`.

## Quoting

A parlay is 2–5 legs. Open a quote request, let market makers price it, then commit the quote you want.

| Endpoint | Purpose |
| --- | --- |
| `POST /v1/quote-requests` | Open a quote request |
| `PATCH /v1/quote-requests/{id}` | Update legs or stake before committing |
| `GET /v1/quote-requests/{id}/quotes` | Fetch current quotes |
| `POST /v1/quote-requests/{id}/commit` | Commit a quote and create the position |
| `POST /v1/quote-requests/{id}/cancel` | Cancel an open request |

```bash
curl -X POST https://api.totalis.trade/v1/quote-requests \
  -H "X-API-Key: <key>" -H "Content-Type: application/json" \
  -d '{
    "legs": [
      { "market_ticker": "KXBTC-25FEB07-T100000", "side": "yes", "venue": "kalshi" },
      { "market_ticker": "0x4d2…",                "side": "no",  "venue": "polymarket" }
    ],
    "bet_amount": 25
  }'
```

Quotes are delivered in real time over SSE. See the [Live Quote Requests](https://docs.totalis.trade/guides/live-quote-requests) and [WebSocket](https://docs.totalis.trade/guides/websocket) guides for the stream contract.

## Funds

| Endpoint | Purpose |
| --- | --- |
| `GET /v1/wallet` | Wallet address and balances |
| `GET /v1/vault` | Vault state and active positions |
| `GET /v1/wallet/delegation-status` | Whether trading is enabled |
| `POST /v1/wallet/enable-trading` | Confirm trading delegation |
| `POST /v1/wallet/withdraw` | Withdraw USDC to an external address |

There is no deposit endpoint. Fund the account by sending native USDC on Solana mainnet to your wallet address; placing a parlay moves the stake into the vault automatically. No SOL is required, as gas is sponsored. The USDC mint is `EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v` (6 decimals).

Withdrawals draw from the vault's free balance — settled winnings and unlocked collateral — to cover the requested amount before sending.

```bash
curl -X POST https://api.totalis.trade/v1/wallet/withdraw \
  -H "X-API-Key: <key>" -H "Content-Type: application/json" \
  -d '{ "recipient": "<solana-address>", "amount_usdc": 50 }'
```
